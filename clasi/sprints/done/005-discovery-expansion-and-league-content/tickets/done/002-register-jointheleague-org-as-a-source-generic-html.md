# Register jointheleague.org as a source (generic_html)

## Description

Register `jointheleague.org` in the Source Registry so League class/
program offerings flow into the directory like any partner's (issue 12).

Live investigation during sprint planning (direct HTTP fetches, not
guesswork) established:
- jointheleague.org is a static site generated by Astro (`<meta
  name="generator" content="Astro v5.16.0">`) — no WordPress, no REST
  API, no TEC signature (`/wp-json/` returns the site's own catch-all
  "not found" page, not a real API payload). **No new adapter type is
  needed** — reuse the existing `generic_html` adapter (sitemap discovery
  + the extraction ladder), same as three of the six currently-registered
  sources.
- Its real sitemap is at `https://www.jointheleague.org/sitemap-index.xml`
  (hyphenated — this is exactly the case ticket 001's `sitemap_url`
  override and parse-based-acceptance fix exist for). It resolves to one
  child, `sitemap-0.xml`, a flat `<urlset>`.
- `sitemap-0.xml` lists `/classes/{slug}/` pages (the League's actual
  offerings — weekly coding classes, camps, workshops) and `/news/
  {yyyy}/{mm}/{dd}/{slug}/` pages (a migrated blog, not offerings).
  `discovery/sitemap.py`'s existing `EVENT_PATH_RE` already includes
  literal `classes` in its alternation and does **not** include `news` —
  confirmed by manually checking the pattern against real sitemap URLs.
  `/classes/*` is included and `/news/*`/`/about/*` are excluded with
  **zero pattern changes**.
- The League already exists in `stem-ecosystem`'s `partners.json` as
  partner id 287, name `"The LEAGUE of Amazing Programmers"` — Normalize's
  existing partner join (`normalize.partners.find_partner`) will match it
  automatically once events with a matching `org_name` flow through; no
  site-repo change needed for the partner attribution itself.
- `robots.txt` (`https://www.jointheleague.org/robots.txt`) is
  `Allow: /` for all user agents — no robots restriction. Its `Sitemap:`
  directive incorrectly points at a `devel.jointheleague.org` staging
  host; this is a bug on the League's own site, not something to fix
  here — do not parse or trust robots.txt's `Sitemap:` line (this
  matches `discovery/sitemap.py`'s existing behavior, which never reads
  it in the first place).

Add `partner_scrape/registry/sources/jointheleague.toml`:
`adapter_type = "generic_html"`, `config.site_url =
"https://www.jointheleague.org"`, `config.sitemap_url =
"https://www.jointheleague.org/sitemap-index.xml"` (ticket 001's
override, set explicitly so this source doesn't depend on probing order
even after the robustness fix), `acquisition_policy.discovered_via =
"sprint 005 planning (live jointheleague.org investigation)"`,
`acquisition_policy.respect_robots = true`.
